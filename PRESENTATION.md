# บทพรีเซนต์ DataPulse

> บทพูดสำหรับนำเสนอ — เหตุการณ์/ตัวเลข/ชื่อไฟล์ ตรงกับข้อมูลตัวอย่างในแอป
> (ดูจุดเล่าเรื่องแบบตารางได้ใน [EXAMPLES.md](EXAMPLES.md))

---

สวัสดีครับ ผมชื่อ **[ชื่อคุณ]** วันนี้ผมจะมานำเสนอเว็บแอปที่ผมสร้างขึ้นด้วย AI-assisted coding ครับ แอปนี้ชื่อว่า **DataPulse**

ก่อนจะอธิบายตัวแอป ผมอยากเริ่มจากปัญหาที่เจอก่อนครับ เวลาข้อมูลจากมิเตอร์หรือเซนเซอร์ออกมา มักเป็นไฟล์ดิบที่มีตัวเลขหลายพันแถว แต่ละอุปกรณ์อาจใช้ชื่อคอลัมน์หรือรูปแบบวันที่ไม่เหมือนกัน พอเปิดใน Excel เราเห็นข้อมูลจำนวนมาก แต่ยังไม่เห็นทันทีว่าค่าแนวโน้มเป็นอย่างไร หรือช่วงไหนมีค่าผิดปกติ

ถ้าต้องมานั่งทำกราฟเอง หาค่าเฉลี่ย สูงสุด ต่ำสุด และไล่หาค่าที่พุ่งผิดปกติทีละแถว ก็ใช้เวลาค่อนข้างมากครับ และคนหน้างานบางคนอาจไม่ได้ถนัด Excel หรือการวิเคราะห์ข้อมูลมากนัก

อีกเรื่องหนึ่งคือข้อมูลพลังงานหรือข้อมูลการผลิตในโรงงาน บางครั้งไม่สะดวกอัปโหลดไปยังระบบภายนอก ผมจึงอยากทำเครื่องมือที่เปิดไฟล์แล้ววิเคราะห์ได้ทันที โดยข้อมูลยังอยู่ในเครื่องของผู้ใช้

เพราะเหตุนี้จึงเกิดเป็น DataPulse ครับ เป็นเว็บแดชบอร์ดสำหรับอ่านและวิเคราะห์ข้อมูลจากมิเตอร์หรือเซนเซอร์ ผู้ใช้เพียงลากไฟล์ `.csv`, `.txt` หรือ `.xlsx` เข้ามา ระบบจะตรวจจับคอลัมน์เวลาและคอลัมน์ตัวเลขให้อัตโนมัติ จากนั้นแสดงกราฟ trend พร้อมสรุปจำนวนข้อมูล ช่วงเวลา ค่าต่ำสุด สูงสุด และค่าเฉลี่ย

ผู้ใช้ยังตั้งค่า Threshold หรือเกณฑ์เตือนได้ เช่น ถ้ากระแสเกินค่าที่กำหนด หรือ PM2.5 เกินเกณฑ์ ให้ไฮไลต์เป็นสีแดงบนกราฟ เพื่อเห็นจุดผิดปกติหรือ anomaly ได้เร็ว โดยเกณฑ์อัตโนมัติที่ผมใช้เป็นแบบ**ทนต่อค่าพุ่งผิดปกติ** ค่าที่เพี้ยนเพียงจุดเดียวจะไม่ทำให้เกณฑ์เพี้ยนตาม นอกจากนี้ยังมี **What-if Simulator** ลองปรับสเกลค่าเพื่อดูผลกรณีสมมติ และยัง**ซูมเข้าไปดูเฉพาะช่วงเวลาที่สนใจ**ได้ด้วย

ตอนนี้ผมจะลองเปิดตัวอย่างข้อมูลให้ดูครับ **[กดปุ่ม "คุณภาพอากาศ กทม." หรือลากไฟล์ `bangkok_air_quality.csv`]**

ตัวอย่างแรกเป็นข้อมูลคุณภาพอากาศรายชั่วโมง 7 วันครับ พอโหลดเข้ามา ระบบจับคอลัมน์ `datetime` เป็นแกนเวลาเอง แล้ววาดกราฟ PM2.5, PM10, อุณหภูมิ และความชื้น พร้อมสรุปค่าให้ทันที

จุดแรก ช่วงบ่ายวันที่ 2 มิถุนายน ค่า PM2.5 พุ่งจากปกติประมาณ 40 ไปถึง **168** ระบบจึงแสดงจุดสีแดงขึ้นมา **[คลิกที่แถวนั้นในตารางจุดผิดปกติ]** พอคลิก กราฟจะ**ซูมและไฮไลต์ไปที่จุดนั้น**ให้เลย ผู้ใช้ไม่ต้องเลื่อนหาเป็นพันแถว ก็รู้ทันทีว่าเหตุการณ์ฝุ่นสูงเกิดวันและเวลาไหน และสังเกตว่า PM10 ก็ขึ้นตาม แปลว่าเป็นฝุ่นจริง สามารถแจ้งเตือนคนในพื้นที่หรือตรวจสอบสาเหตุต่อได้เร็วขึ้นครับ

จุดที่สอง วันที่ 4 อุณหภูมิขึ้นไป 46 องศาเพียงจุดเดียว แล้วกลับสู่ค่าปกติทันที ลักษณะนี้อาจไม่ใช่อุณหภูมิจริง แต่เป็นความคลาดเคลื่อนจากเซนเซอร์ หรือ sensor glitch และถ้าดูช่องความชื้นก็ไม่ได้เปลี่ยนตาม ยิ่งยืนยันว่าเป็นค่าที่ผิด ประโยชน์คือช่วยคัดแยกข้อมูลที่น่าสงสัยออกก่อนนำไปคำนวณ เพราะถ้าเอาค่าผิดนี้ไปหาค่าเฉลี่ย อาจทำให้ตัดสินใจผิดได้ครับ

จุดที่สาม วันที่ 5 ช่วงบ่าย จะเห็นว่า PM2.5 และ PM10 ตกฮวบ พร้อมกับความชื้นที่พุ่งขึ้นเกือบ 100% และอุณหภูมิที่ลดลง **[กดปุ่ม "ปรับสเกล" เพื่อให้ทุกเส้นเทียบกันได้]** จะเห็นชัดเลยว่าหลายค่าขยับสัมพันธ์กัน — นี่คือช่วงที่ฝนตกครับ ฝนชะล้างฝุ่น ค่าฝุ่นที่ต่ำผิดปกติจึงมีเหตุผล ไม่ใช่เซนเซอร์พัง จุดนี้แสดงให้เห็นว่าแดชบอร์ดช่วย**อ่านความสัมพันธ์ระหว่างค่าหลายตัว** ไม่ใช่แค่ดูทีละเส้น

ต่อไปเป็นตัวอย่างข้อมูลไฟฟ้าครับ **[กดปุ่ม "มิเตอร์ไฟฟ้า" หรือลากไฟล์ `electricity_meter.csv`]**

จุดแรก กระแสพุ่งจาก 13 แอมป์ไปถึง **56 แอมป์** ในช่วงเวลาเดียว ระบบแสดงเป็น anomaly ให้เห็นชัด กรณีนี้อาจเกิดจากมอเตอร์เริ่มทำงานหรือมีโหลดใหญ่ต่อเข้าระบบ ช่วยให้ช่างตรวจสอบได้เร็วว่ากระแสกระชากเกิดเมื่อไรและบ่อยแค่ไหน เพื่อป้องกันเบรกเกอร์ตัด สายไฟร้อน หรืออุปกรณ์เสียหาย

จุดที่สอง แรงดันไฟตกจากประมาณ 230 โวลต์ เหลือ **196 โวลต์ต่อเนื่องประมาณหนึ่งชั่วโมง** ซึ่งเรียกว่า brownout **[ลากเลือกช่วงนั้นบนกราฟเพื่อซูมเข้าไป]** พอซูมเข้าไป แกนค่าจะขยายให้เห็นรายละเอียดชัดเจนว่า ไม่ได้เป็นค่าตกเพียงจุดเดียว แต่เป็นปัญหาต่อเนื่องที่อาจส่งผลต่อมอเตอร์ เครื่องจักร หรืออุปกรณ์อิเล็กทรอนิกส์ ทำให้ทีมงานนำข้อมูลไปตรวจสอบแหล่งจ่ายไฟต่อได้ตรงจุด

จุดที่สาม วันที่ 3 ช่วงบ่าย มีโหลดใหญ่ทำงาน จะเห็นว่ากระแสขึ้น กำลังไฟขึ้น แรงดันตกเล็กน้อย และค่า Power Factor ลดลง พร้อมกันทั้งหมด นี่คืออีกตัวอย่างที่หลายค่าสัมพันธ์กัน บอกได้ว่ามีโหลดใหญ่ต่อเข้ามา ต่างจากมอเตอร์สตาร์ทที่กระชากแวบเดียวครับ

นอกจากนี้ยังมี What-if Simulator **[เลื่อนสไลเดอร์ปรับสเกลค่า]** ลองสมมติว่าโหลดเพิ่มขึ้น แล้วดูว่าจำนวนจุดที่เกินเกณฑ์เปลี่ยนไปอย่างไร และผู้ใช้ยัง**ออกรายงานสรุปเป็น PDF** ได้ในคลิกเดียวเพื่อแนบเอกสารหรือส่งต่อ โดยข้อมูลทั้งหมดประมวลผลในเบราว์เซอร์ ไม่ถูกอัปโหลดออกไปไหนเลยครับ

ในส่วนของการพัฒนา ผมใช้ **Claude Code** ซึ่งเป็น AI Coding Assistant โดยเป็นแนวทาง low-code หรือ vibe coding ครับ ผมไม่ได้เขียนโค้ดทุกบรรทัดด้วยตัวเอง แต่บอกความต้องการเป็นภาษาธรรมดา แล้ว AI ช่วยสร้างโค้ด รันทดสอบ และช่วยแก้ปัญหาระหว่างพัฒนา โดยผมเป็นคนกำหนดฟังก์ชัน ตรวจสอบผลลัพธ์ และปรับงานให้ตรงกับการใช้งานจริง

ตัวแอปเขียนด้วยภาษา **TypeScript** ซึ่งเป็น JavaScript ที่มีระบบ type ช่วยจับข้อผิดพลาดได้ตั้งแต่ตอนพัฒนา หน้าเว็บใช้ JSX และ HTML ส่วนการออกแบบใช้ CSS ครับ

แพลตฟอร์มหลักคือ **Next.js 14** และ **React** รันผ่าน **Node.js** ใช้ **Recharts** สำหรับสร้างกราฟ ใช้ **PapaParse** สำหรับอ่านไฟล์ CSV และ TXT และใช้ **SheetJS** สำหรับอ่านไฟล์ Excel โดยเก็บ source code ไว้บน GitHub

Prompt หลักที่ใช้สั่ง AI เริ่มต้นมีลักษณะประมาณนี้ครับ

```
สร้างเว็บแอป Next.js + TypeScript ชื่อ DataPulse
เป็น dashboard ที่ลากไฟล์ .csv/.txt/.xlsx เข้ามาแล้ว:
- auto-detect คอลัมน์เวลากับคอลัมน์ตัวเลขเอง
- โชว์ KPI จำนวนแถว ช่วงเวลา min max avg
- วาดกราฟ trend หลายเส้น
- ตั้ง threshold แล้วไฮไลต์จุดที่เกินเกณฑ์เป็นสีแดง
- มี simulator ปรับสเกลค่าแบบ what-if
ทำงานฝั่งเบราว์เซอร์ล้วน ไม่มี backend และไม่ใช้ localStorage
```

หลังจากนั้นผมค่อยสั่งปรับเพิ่มแบบคุยโต้ตอบ เช่น เปลี่ยนธีมให้เป็นโทนสว่างโมเดิร์น เพิ่มการซูมกราฟและคลิกจุดผิดปกติแล้วกระโดดไปดู ออกรายงาน PDF และปรับชุดข้อมูลตัวอย่างให้เหมาะกับการนำเสนอ จนได้ DataPulse เวอร์ชันนี้ครับ

สรุปคือ DataPulse ช่วยเปลี่ยนข้อมูลดิบจากมิเตอร์และเซนเซอร์ ให้กลายเป็นข้อมูลที่อ่านง่าย เห็นแนวโน้ม ตรวจพบความผิดปกติ และนำไปตัดสินใจหน้างานได้เร็วขึ้น โดยข้อมูลไม่ถูกอัปโหลดออกจากเครื่องของผู้ใช้ครับ ขอบคุณครับ
