# คู่มือพรีเซนต์ — ชุดข้อมูลตัวอย่าง

ชุดข้อมูลทั้งสองถูกออกแบบให้มี "จุดเล่าเรื่อง" ชัดเจน สำหรับเดโม DataPulse
โหลดได้จากปุ่มในแอป (**คุณภาพอากาศ กทม.** / **มิเตอร์ไฟฟ้า**) หรือลากไฟล์จาก
`public/examples/` เข้ามาเอง · สร้างใหม่ได้ด้วย `node scripts/generate-examples.mjs`

> เกณฑ์อัตโนมัติใช้ **median ± MAD** (ทนต่อ outlier) — สไปก์ "sensor บัค" ตัวเดียว
> จะไม่ดันเกณฑ์ให้เพี้ยน จุดผิดปกติจริงเลยยังถูกจับครบ

---

## 1) คุณภาพอากาศ กทม. — `bangkok-air-quality.csv`
รายชั่วโมง 7 วัน (จ. 12 – อา. 18 ม.ค. 2026) · คอลัมน์: `datetime, PM2.5, PM10, O3, NO2, Temp, Humidity`

### จุดที่ระบบจับให้เลย (auto-threshold)
| เวลา | สิ่งที่เกิด | พูดว่า |
| --- | --- | --- |
| **17 ม.ค. 14:00** | PM2.5 = **320** ขณะ PM10 ปกติ (~42) | "นี่ **sensor บัค** — ค่าเด้งผิดธรรมชาติเฉพาะช่อง PM2.5 ถ้าฝุ่นจริง PM10 ต้องขึ้นตาม" |
| ทุกวัน **12:00–15:00** | O3 สูง | "กลางวันแดดจัด เกิดโอโซนจากปฏิกิริยาแสง" |
| ทุกวัน **07–09 / 17–20** | NO2 สูง | "ชั่วโมงเร่งด่วน เช้า-เย็น รถติด ไอเสียพุ่ง" |

### จุดที่ตั้งเกณฑ์เองเพื่อโชว์ (เดโม interactive)
- ตั้ง **PM2.5 max = 50** → ทุกเย็น 17–20 และ **15–16 ม.ค. ทั้งวัน** ติดแดง
  → "**ฝุ่นเยอะตอนคนเลิกงาน** และช่วง 2 วันนี้อากาศปิด/หมอกควันต่อเนื่อง"
- **18 ม.ค. 03:00–06:00** PM2.5 ค้างนิ่งที่ **11.0** เป๊ะหลายชั่วโมง
  → "**sensor ค้าง/ออฟไลน์** — ค่าไม่ขยับเลย" (เห็นได้จากกราฟ/ตาราง)

🎯 **เดโมเด็ด:** คลิกแถว PM2.5 = 320 ในตาราง → กราฟไฮไลต์จุดนั้น เห็นเลยว่าโดดออกมาขนาดไหน

## 2) มิเตอร์ไฟฟ้า — `power-meter.csv`
ทุก 15 นาที 3 วัน (จ. 2 – พ. 4 มี.ค. 2026) · คอลัมน์: `Voltage, Current, Power_kW, PowerFactor, Frequency, Energy`

### วงจรปกติ (เห็นจากรูปกราฟ ไม่ใช่ anomaly)
กลางคืน 00–06 โหลดต่ำ → เวลาทำการ 08–17 โหลดสูงคงที่ → เย็น 18–21 บัมพ์ (คนกลับบ้าน เปิดแอร์/ไฟ)

### จุดที่ระบบจับให้เลย (auto-threshold)
| เวลา | สิ่งที่เกิด | พูดว่า |
| --- | --- | --- |
| **3 มี.ค. 10:30** | Current = **210 A**, Power ~43 kW | "**มิเตอร์อ่านพลาด/บัค** ค่าพุ่งเว่อร์" |
| **2 มี.ค. 19:00** | Voltage ตกเหลือ **212 V** | "**ไฟตก**ชั่วครู่ตอนโหลดพีคเย็น" |
| **4 มี.ค. 02:00–02:15** | ทุกค่าตก (Voltage ~8 V) | "**ไฟดับ**สั้นๆ กลางดึก" |

### จุดที่ตั้งเกณฑ์เอง / ใช้ simulator
- **2 มี.ค. 08:00** ไฟกระชาก (Current สูงชั่ววูบ) → "เครื่องจักรใหญ่สตาร์ทตอนเช้า"
- ตั้ง **Current max = 45** → ช่วงเวลาทำการ + เย็นติดแดง → "ช่วงโหลดหนักของวัน"

🎯 **เดโมเด็ด:** เปิด **Simulator** เลื่อน Current ×1.2 → ดูจำนวนจุดเกินเกณฑ์เพิ่มขึ้น
= "ถ้าโหลดโตอีก 20% จะเกินพิกัดกี่จุด"

## โฟลว์เดโมแนะนำ (2–3 นาที)
1. กดปุ่มชุดข้อมูล → ดูกราฟ "วาดตัวเอง" + KPI ขึ้นทันที
2. ชี้สไปก์ sensor บัค → คลิกแถวในตาราง → กราฟไฮไลต์ให้
3. แก้ threshold เป็นค่ามาตรฐาน → จุดผิดปกติอัปเดตเรียลไทม์ (เล่าเรื่อง rush hour / โหลดหนัก)
4. สลับโหมด **ปรับสเกล** → เทียบหลายเส้นที่สเกลต่างกันได้
5. เปิด **Simulator** → จำลอง what-if
